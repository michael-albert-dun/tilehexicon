# Generation Order Notes

These notes record provisional decisions about how TileHexicon generates and
recognises words on polyhex pieces. They are intentionally a working log rather
than final rules.

## General Principle

Puzzle generation should use fixed, natural reading orders for each shape
orientation. Solving can be more permissive: a player solution may be accepted
as an anagram if it forms a legal piece, even if that order would not have been
used to generate the puzzle.

This split lets generated puzzles look fair and readable while still allowing
players to discover legitimate alternate groupings.

## Tetrahexes

- Fixed tetrahex orientations are listed in `data/tetrahex-shapes.txt`.
- Explicit generation orders are stored in `data/tetrahex-generation-orders.json`.
- Shapes not explicitly listed use normalized shape order.
- The review helper was previously `generation-order-review.html`; if restored,
  it should show fixed tetrahex orientations with their allowed generation
  orders.

Known issue: because solving accepts anagrams, a board can contain alternate
valid-looking groupings that were not generated. Possible future responses:

- Filter generated boards that contain too many tempting alternate valid words.
- Add a strict solving mode that accepts only generation-order readings.
- Keep current liberal solving as the default but surface the ambiguity in the
  instructions.

## Penthexes

Penthex shape data is in `data/penthex-shapes.txt`. The review helper is
`docs/penthex-generation-order-review.html`.

The canonical association table is now:

```text
data/penthex-generation-orders.json
```

It maps each 1-based fixed penthex shape index to one or more compact generation
order strings, such as `"12345"` or `"12453"`. This is the table game/puzzle
generation should use. It is generated by:

```sh
scripts/generate-penthex-generation-orders.py
```

For review purposes, some penthex orientations are omitted because their order is
currently considered unambiguous. These filters are chained in order:

- If the adjacency graph is a single path of five cells, read along the path
  from the earlier top/left endpoint. In raw shape-index notation this is not
  necessarily `1-2-3-4-5`.
- From the remaining shapes, if there is exactly one Hamiltonian path whose
  steps all go rightward or downward, that path is standard and the shape can be
  omitted from manual review.
- From the remaining shapes, if there is exactly one Hamiltonian path that uses
  at most two of the allowed move types, that path is standard and the shape can
  be omitted from manual review. The current allowed move types are `RU`, `RH`,
  `RD`, and `LD`.
- From the remaining shapes, if there is exactly one legal Hamiltonian path that
  always goes horizontally or downward, that path is standard and the shape can
  be omitted from manual review. This currently allows `RH`, `RD`, and `LD`.
- From the remaining shapes, if exactly two legal Hamiltonian paths remain and
  the first step of one goes rightward while the first step of the other goes
  leftward, the rightward-starting path is standard and the shape can be omitted
  from manual review.
- From the remaining shapes, if there is exactly one Hamiltonian path whose
  steps always go rightward, that path is standard and the shape can be omitted
  from manual review.
- From the remaining shapes, if the shape has height four, read top to bottom,
  and left to right within each level. The shape can be omitted from manual
  review.

For shapes that remain in review, the first-pass suggested order prefers
Hamiltonian paths using only the allowed move types whenever such paths exist.
The geometric scoring heuristic is used only to choose among those legal paths,
or among all Hamiltonian paths if there are no legal paths.

Current counts:

- 186 fixed penthex orientations.
- 54 single-path orientations.
- 36 unique right/down path orientations.
- 7 unique two-move-type path orientations.
- 2 unique horizontal/down legal path orientations.
- 1 two-legal-path orientation resolved by the rightward first-step preference.
- 1 unique always-rightward path orientation.
- 8 height-four orientations.
- 77 orientations are resolved manually and included in the canonical table.

These counts are based on the current generated shape list and may change if the
shape ordering or filter definitions change.

The canonical penthex generation-order table is generated by
`scripts/generate-penthex-generation-orders.py` and written to
`data/penthex-generation-orders.json`. The letter-order review page uses `LUNGE`
as a display word and lets manual changes be discussed as readings such as
`37: Add LUNEG`; these are translated back to raw cell-index orders in the
script.

## Radius-2 Tetrahex/Penthex Tilings

The generator supports mixed exact covers. For example:

```sh
scripts/generate-polyhex-tilings.py --radius 2 --holes 0 --piece-orders 4,5,5,5
```

Each line is a 19-character board string in radius-cell order sorted by `(r, q)`.
For mixed files, labels follow the `--piece-orders` list. For example, in
`--piece-orders 4,5,5,5`, label `0` is the tetrahex and labels `1`, `2`, and
`3` are penthexes. Pieces are not forced to cover the first board cell in order;
any remaining piece type may cover the first empty cell during search.

Current radius-2 tiling counts:

- `data/tetrahex-tilings-radius-2-holes-3.txt`: 9,628 tilings with four
  tetrahexes and three holes.
- `data/polyhex-tilings-radius-2-orders-4-4-4-5-holes-2.txt`: 13,520 tilings
  with three tetrahexes, one penthex, and two holes.
- `data/polyhex-tilings-radius-2-orders-4-4-5-5-holes-1.txt`: 4,695 tilings
  with two tetrahexes, two penthexes, and one hole.
- `data/polyhex-tilings-radius-2-orders-4-5-5-5-holes-0.txt`: 364 tilings with
  one tetrahex and three penthexes.

The helper `docs/mixed-tilings-preview.html` shows the no-hole mixed tilings visually.

Random puzzle generation should not choose uniformly from the union of these
files, because the two- and three-penthex cases would be underrepresented. The
current plan is:

1. Choose the number of penthexes uniformly from `0, 1, 2, 3`.
2. Choose uniformly from the corresponding tiling file.

## Things To Revisit

- Whether liberal anagram solving is too permissive for penthex puzzles.
- Whether generation should reject boards with plausible alternate solutions.
- Whether the current allowed move types `RU`, `RH`, `RD`, and `LD` are the right
  basis for defining standard penthex paths.
- Whether the 4-letter allowed word list should be cut down. The rough review
  ranking is generated by `scripts/rank-four-letter-candidates.py` and written to
  `data/allowed-words-4-review-ranked.txt`.
